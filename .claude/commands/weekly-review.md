Weekly review mode — the "review" session type defined in `CLAUDE.md`. This
runs everything in a standup, plus review-specific steps.

Steps:

1. Read `FACTS.md`, `SPRINT.md`, `LOG.md`, `INBOX.md`, and `LEARNINGS.md` —
   reviews are one of the times `LEARNINGS.md` should be read, per CLAUDE.md's
   context rules. Don't read `archive/` unless something makes it relevant.
2. Run the standup flow first: process `INBOX.md` item by item (confirm
   routing with me before filing each one), then ask what happened
   yesterday/this week and what's blocking, one clarifying question at a time.
3. Score the week: tasks completed vs. planned, Japanese study minutes,
   outreach sent, replies received, events attended. Ask me for any numbers
   that aren't already logged — don't guess them.
4. Talk through what worked (keep doing it) and what didn't (propose a
   specific change and get my explicit agreement before treating it as
   decided).
5. Plan next sprint's backlog in `SPRINT.md`.
6. Remind me to check the next 2 weeks for holidays, family commitments, and
   my wife's evening schedule BEFORE committing to any networking events.
7. Update `LEARNINGS.md` with the week's insights.
8. Prune `LOG.md` entries older than 14 days — they already live in
   `archive/`.
9. Don't file anything until I say **"wrap"** — then run the wrap pipeline
   defined in `CLAUDE.md`.

This is meant for Fridays. If I run it on another day, don't refuse — just
note the mismatch and continue.
